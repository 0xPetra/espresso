CryptoError

Fields

err: Result<TxnApiError, String>

An error from the Jellyfish library

Note: This is used to wrap [TxnApiError] because it cannot be serialized. TxnApiError cannot be serialized because it depends on many foreign error types which do not implement the Serialize trait. Instead, if we have to serialize this variant, we will serialize Ok(err) to Err(format(err)), and when we deserialize we will at least preserve the variant CryptoError and a String representation of the underlying error.

UnsupportedTransferSize

Fields

num_inputs: usize
num_outputs: usize

The transfer transaction has an unsupported number of inputs or outputs.

Note: For transactions with fewer inputs or outputs than supported, the transaction should be padded with dummy transactions. If transactions with a greater number of inputs or outputs are required, then the universal parameter set for the ledger must be updated.
